# Пользовательские DOCX-шаблоны

С версии с поддержкой шаблонов можно передать собственный `.docx` при генерации ТК+МК.

> В репозитории эталон `templates/tk_default.docx` хранится в текстовом WordprocessingML-формате (без бинарного zip), чтобы избежать проблем с бинарными файлами. Движок поддерживает оба варианта: обычный DOCX-архив и XML-шаблон.

## Запуск

```bash
tk-generator --input examples/product_minimal.json --output output/ --template ./templates/tk_default.docx
```

Если флаг `--template` **не указан**, используется текущий встроенный генератор (обратная совместимость).

## Поддерживаемые плейсхолдеры

- `{{product.name}}` — наименование изделия.
- `{{operations_table}}` — таблица операций (раздел 6).
- `{{mk_table}}` — таблица маршрутной карты (МК).

Также доступны плоские поля объекта `product` в формате dot-notation, например:

- `{{product.tk_number}}`
- `{{product.dimensions.length}}`
- `{{product.material.type}}`

## Как работает движок

`src/template-engine.js`:

1. Пытается открыть `.docx` как zip-архив; если это текстовый WordprocessingML, собирает минимальный DOCX-пакет автоматически.
2. Читает `word/document.xml`.
3. Подставляет значения `{{...}}` из входных данных.
4. Заменяет абзацы-плейсхолдеры `{{operations_table}}` и `{{mk_table}}` на XML-таблицы.
5. Сохраняет новый DOCX.

## Рекомендации по шаблону

- Размещайте `{{operations_table}}` и `{{mk_table}}` в отдельных абзацах.
- Не разбивайте плейсхолдер на несколько run-сегментов вручную в редакторе Word.
- В качестве базы можно использовать `templates/tk_default.docx`.
