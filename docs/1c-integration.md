# Интеграция с 1С

## Назначение

Экспорт уже рассчитанных калькуляций в XML/CSV для загрузки в 1С-процессы.

## Реально поддерживаемое

- `--export-1c` => `output/1c_export_<YYYY-MM-DD>.xml`
- `--export-1c-csv` => `output/1c_export_<YYYY-MM-DD>.csv`

Флаги можно использовать вместе.

## Пошаговый пример

```bash
npx tk-generator \
  --input examples/batch_small.json \
  --output output/ \
  --export-1c \
  --export-1c-csv
```

## Какие поля выгружаются

- Номенклатура: наименование, артикул (`tk_number`), единица.
- Калькуляция: материалы, ФОТ, накладные, прибыль, итог.
- Спецификация: операции с нормами труда/машинного времени.

## Ограничения (по текущему коду)

- Есть только экспорт XML/CSV.
- Прямой online-обмен с 1С (HTTP/OData/COM) в проекте не реализован.

## Troubleshooting

- Если экспорт не появился — проверьте путь `--output` и права записи.
- Экспорт строится на данных расчёта себестоимости; невалидный вход нужно исправить до экспорта.
